# How can you adjust your communication style to be more inclusive of neurodivergent users and teammates?

I can make my communication much more literal, structured, and clear. For teammates, this means writing concise pull request descriptions, breaking down complex tasks into numbered steps, and avoiding ambiguous language or sarcasm in text channels like Slack. For users, it means ensuring error messages and documentation clearly state what went wrong and exactly how to fix it, without relying on implied knowledge.

# What are some common UX or communication pitfalls that might make Focus Bear less accessible or supportive?

A major UX pitfall is "information overload", which presents a user with a massive wall of text or too many configuration options on a single screen. For someone having difficulty in digesting these things, having to make 10 decisions just to start a timer can cause paralysis. Another communication pitfall is relying solely on color (like red for errors) without text labels, or sending notifications that demand immediate, complex action without clear instructions.

# What is one practical change you can make in your work to better support vulnerable populations?

As a developer, I can strictly commit to using semantic and proper labels when building user interfaces. This ensures that any frontend components I create are fully compatible with screen readers and other assistive technologies, making the application more accessible to everyone regardless of how they navigate the web.

# Insight from First-Person Accounts

Based on accounts from users with ADHD, a common barrier is the "wall of awful" or executive dysfunction, where the friction of starting a task prevents it from happening at all. Productivity tools often fail when they require too much upfront planning before delivering value.

# Identify one design or communication improvement that could make Focus Bear more accessible.

We could implement "One-Click Quick Starts" or pre-built routine templates during the onboarding process. Instead of forcing a new user to build their morning routine from scratch (which requires high executive function), giving them a default template they can immediately use lowers the friction to getting started.
