# Reflection

### How does Kanban help manage priorities and avoid overload?

Kanban provides a clear visual hierarchy. By looking at the "Backlog" or "Next" column, I can instantly see which technical tasks have the highest priority assigned by my lead. It prevents overload by using "Work In Progress" limits. Instead of feeling overwhelmed by the entire project, I only focus on the 1 or 2 tasks currently in my "In Progress" column.

### How can you improve your workflow using Kanban principles?

I can improve my workflow by being more disciplined with the "Blocked" column. Instead of just stopping work when I hit a technical hurdle, I will move the task to "Blocked" and clearly comment on the GitHub card exactly what is stopping me. This allows my teammates to see I need help without me having to chase them down on Discord or on email.

# Task

### Identify one way you can improve task tracking in your role.

I will improve task tracking by breaking down large GitHub issues into smaller, manageable "sub-tasks" on the Kanban board. This will give me a better sense of daily progress and provide more granular updates for my weekly reports.
